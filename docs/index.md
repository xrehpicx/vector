# Vector Documentation

This documentation covers the current development workflow for Vector. If you are contributing to the repository, start here before diving into feature work.

## Documentation Sections

The docs are organized by the part of the project they explain.

### 1. [Getting Started](./getting-started/01-local-setup.md)

This section covers local setup, environment variables, and everyday development commands.

### 2. [Product](./product/01-overview.md)

This section explains the product, who it is for, and the main workflows it supports.

### 3. [Architecture](./architecture/01-stack.md)

This section gives a high-level view of the stack, project structure, permissions model, and storage flow.

### 4. [Development](./development/01-coding-style.md)

This section contains developer conventions for coding style, routing, type safety, component usage, and error handling.

### 5. [Design](./design/01-design-system.md)

This section outlines the UI patterns and design principles used across the app.

### 6. [Architectural Decision Records (ADR)](./adr/001-service-layer-implementation-plan.md)

This section records major technical decisions and the reasoning behind them.

---

## Notes

- Prefer the getting-started guides and top-level repository docs for onboarding.
- The `docs/migration/` directory is historical reference material from earlier architecture work and may not match the current contributor setup.
- Some plan and implementation-summary files under `docs/development/` are historical notes rather than onboarding docs. Prefer the numbered guides in each section first.
- For contribution workflow, see [CONTRIBUTING.md](../CONTRIBUTING.md).
